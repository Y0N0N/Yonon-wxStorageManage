# 🎯 Haili 物资管理小程序 — 项目目标

> 本文档定义本小程序的**产品定位、目标用户、核心功能、技术架构**，所有开发决策须以此为基准。

---

## 一、产品定位

### 产品名称
**Haili** — 办公室物资管理助手

### 一句话描述
一个面向高校/行政办公室的**轻量级物资管理工具**，让帐篷、桌子、胶凳等物资的进出库管理透明、可追溯、零混乱。

### 产品愿景
取代纸质登记台账，实现物资全生命周期数字化管理，做到：
- **去向可查** — 每件物资谁借的、谁还的、什么时候、多少数量，一目了然
- **状态可见** — 当前库存、借出中、待归还，看板式直观展示
- **操作可溯** — 每一次编辑、借出、归还都有完整的操作日志

---

## 二、目标用户

### 第一类：办公室管理员 👤
- 角色：办公室正式工作人员 / 老师
- 权限：**物资只读 + 账号管理**
  - 查看所有物资的实时库存
  - 查看所有借出/归还记录
  - 查看操作日志
  - 查看统计报表
  - **创建学生助理账号**
  - **编辑学生助理账号**
  - **删除学生助理账号**
- 主要承担**监督审核与账号管理**职能

### 第二类：学生助理 👥
- 角色：勤工助学学生 / 兼职助理
- 权限：**仅操作物资，不可管理账号**
  - 借出物资（记录借用人、负责人、数量、时间）
  - 归还物资（记录归还时间、接收人）
  - 编辑物资信息（新增/修改/删除物资，每次自动记录）
- 是系统的**日常操作用户**，所有操作留痕
- **不可创建、编辑、删除任何用户账号**

### 权限对比

| 功能 | 管理员 | 学生助理 |
|------|:------:|:--------:|
| 登录 | ✅ | ✅ |
| 查看物资看板 | ✅ | ✅ |
| 查看借出/归还记录 | ✅ | ✅ |
| 查看操作日志 | ✅ | ✅ |
| 借出物资 | ❌ | ✅ |
| 归还物资 | ❌ | ✅ |
| 新增/编辑/删除物资 | ❌ | ✅ |
| 创建学生助理账号 | ✅ | ❌ |
| 编辑学生助理账号 | ✅ | ❌ |
| 删除学生助理账号 | ✅ | ❌ |
| 查看统计报表 | ✅ | ✅ |

## 三、核心功能

### 3.1 身份认证与权限控制
- 仅**管理员**与**学生助理**两类账号可登录
- 登录方式：微信授权 + 自定义账号体系
- 后端鉴权：每次请求校验角色身份
- 未登录用户不可访问任何页面

### 3.2 物资看板（首页 / Dashboard）
直观展示以下核心数据卡片：
- 📦 **在库物资** — 当前仓库中实有数量（按类别统计）
- 📤 **已借出** — 当前所有被借走的物资及数量
- ⏳ **待归还** — 超过预计归还时间仍未归还的物品（逾期警示）
- 🕐 **最近动态** — 最近 10 条操作记录（借出/归还/编辑）

> 管理员和助理看到的看板内容一致，但助理界面上多出"借出/归还/编辑"的操作入口。

### 3.3 物资目录管理
管理全部物资品类，每个物资包含：
- 名称（如：帐篷、折叠桌、胶凳）
- 类别（如：户外设备、桌椅、其他）
- 规格/备注（如：3m×3m、绿色）
- 当前库存总量
- 在库数量（= 总量 − 已借出）
- 创建时间、最后修改时间

### 3.4 借出管理
执行流程：
1. 选择物资 → 填写借出表单
2. 表单**必填字段**：
   - 物资名称（选择）
   - 借出数量
   - **借用人**（谁借的，如某部门/个人）
   - **负责人**（审批/负责的老师）
   - **学生助理**（执行本次操作的人 → 自动获取当前登录用户）
   - 预计归还时间
   - 备注（可选）
3. 提交后：
   - 该物资在库数量减少
   - 生成一条"借出记录"
   - 更新看板数据

### 3.5 归还管理
执行流程：
1. 查看"已借出"列表 → 选择要归还的记录
2. 填写归还表单：
   - 实际归还数量（支持部分归还）
   - **接收人**（谁接收的）
   - 备注（可选）
3. 提交后：
   - 该物资在库数量增加
   - 该借出记录状态更新（已归还 / 部分归还）
   - 生成一条"归还记录"
   - 更新看板数据

### 3.6 物资编辑（仅学生助理）
- **新增物资**：录入名称、类别、数量、规格
- **修改物资**：调整数量、名称、类别等信息
- **删除物资**：二次确认后方可删除
- ⚠️ **关键约束**：每次编辑自动生成**操作日志**，记录：
  - 操作人（学生助理）
  - 操作类型（新增/修改/删除）
  - 操作时间
  - 操作前快照（修改/删除时）
  - 操作后快照（新增/修改时）

### 3.7 操作日志
所有操作的统一归档，支持按时间、操作人、操作类型筛选。

| 字段 | 说明 |
|------|------|
| 操作人 | 执行操作的学生助理姓名 |
| 操作类型 | 借出 / 归还 / 新增物资 / 修改物资 / 删除物资 |
| 关联物资 | 操作涉及的物资名称 |
| 数量变化 | +N / -N / 修改前后对比 |
| 负责人/借用人 | 借出/归还时关联的人员 |
| 时间 | 精确到秒的操作时间 |
| 操作前快照 | 编辑类操作的数据原始值 |
| 操作后快照 | 编辑类操作的数据新值 |

## 四、数据模型概览（核心实体）

### 4.1 用户（User）
```typescript
interface User {
  id: string;
  name: string;
  role: 'admin' | 'assistant';
  phone?: string;
  createdAt: string;
  isActive: boolean;
}
```

### 4.2 物资（Material）
```typescript
interface Material {
  id: string;
  name: string;           // 物资名称（如"帐篷"）
  category: string;       // 物资类别
  spec?: string;          // 规格/备注
  totalQuantity: number;  // 总数量
  currentStock: number;   // 当前在库数量
  unit: string;           // 单位（个/张/把）
  imageUrl?: string;      // 物资图片
  createdAt: string;
  updatedAt: string;
}
```

### 4.3 借出记录（BorrowRecord）
```typescript
interface BorrowRecord {
  id: string;
  materialId: string;
  materialName: string;
  quantity: number;
  borrower: string;       // 借用人
  responsiblePerson: string; // 负责人
  assistantName: string;  // 学生助理
  borrowTime: string;
  expectedReturnTime: string;
  status: 'borrowed' | 'partial' | 'returned';
  returnedQuantity: number;
  actualReturnTime?: string;
  receiver?: string;      // 归还接收人
  note?: string;
  createdAt: string;
}
```

### 4.4 操作日志（OperationLog）
```typescript
interface OperationLog {
  id: string;
  operatorName: string;   // 操作人（学生助理）
  operationType: 'borrow' | 'return' | 'create' | 'update' | 'delete';
  materialId: string;
  materialName: string;
  quantityChange?: number; // 数量变化
  relatedPerson?: string;  // 关联的借用/负责人
  beforeSnapshot?: object; // 操作前快照
  afterSnapshot?: object;  // 操作后快照
  timestamp: string;
  detail: string;          // 可读描述
}
```

## 五、技术架构

### 前端
- 平台：**微信小程序**
- 语言：**TypeScript**
- 样式：**Sass**（编译为 `.wxss`）
- 渲染引擎：**Skyline**
- 组件框架：**glass-easel**

### 后端（待定，建议方案）
| 方案 | 适用阶段 | 说明 |
|------|----------|------|
| 微信云开发 | MVP / 原型 | 云数据库 + 云函数，免运维 |
| 自建后端 | 正式上线 | 提供更灵活的权限控制和数据量支持 |

### 数据存储
- 用户数据、物资数据、操作日志存储在数据库中
- 关键字段建立索引（按时间、角色、物资ID）

### 认证流程
```
微信登录 → 获取 openid → 查询用户表 → 校验角色 → 下发 token
                                 ↓
                          无权限 → 拒绝访问
```

## 七、设计原则

1. **极简操作** — 借出/归还流程不超过 3 步
2. **数据透明** — 每件物资的状态变化均可追溯
3. **权限明确** — 管理员只读、助理可写，职责分离
4. **离线可用** — 网络不佳时保留核心操作能力（待实现）
5. **移动优先** — 基于小程序，随时随地可操作
